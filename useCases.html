Metadata Use Cases

Metadata are created and used in many different communities which support their own metadata standards, repositories, and infrastructures. Many of these communities have similar needs, and they support many of the same use cases to address those needs. For example, essentially all repositories, regardless of the metadata standard they use, support resource discovery, some form of access and the metadata support those processes. As these standards all support the same conceptual documentation language, we call them metadata dialects.

The DataCite metadata schema has existed since 2011 with the goal of facilitating identification and citation of datasets and other kinds of research outputs (resources). The schema has evolved considerably throughout its history and can now support other important metadata use cases. In particular, a recommendation for how the DataCite metadata schema can support all of the FAIR use cases (Findability, Accessibility, Interoperability, and Re-use) was proposed several years ago and has been used to evaluate metadata in many repositories all over the world (Bright Spots get Brighter, A PID Feast for Research - PIDapalooza 2021).

This page lists the metadata use cases used by the Metadata Game Changers completeness tools, and the DataCite metadata paths each concept is drawn from. Background is available in:

  • MetaDIG recommendations for FAIR DataCite metadata
  • Minimum Metadata
  • A PID Feast for Research – PIDapalooza 2021
  • FAIR DataCite Metadata - University and College Bright Spots
  • DataCite Bright Spots
  • Bright Spots get Brighter
  • DataCite Bright Spots - Repositories, Consortia, and Improvements
  • Community Can Help Improve Metadata
  • DataCite Project Use Cases

Note: these results measure completeness with respect to a known recommendation. They should not be construed as a measure of metadata quality. They could be a small part of an assessment of metadata quality, but we should be careful not to use that term to describe them.
